# 晨昏之线

时间感知与整点主动问候插件，适用于 [my-neuro](https://github.com/A-night-owl-Rabbit/my-neuro) 框架。

**当前版本：1.1.0**（与 `metadata.json` / `package.json` 一致）

## 功能

### 环境感知（对齐 LLM 场景感知思路）

- 可选在**每次大模型请求**时，于最后一条用户消息前注入一行前缀：`[发送时间: … | 星期, 工作日/法定节假日/调休, 上午|中午|下午|晚上|深夜]`，便于模型感知当前场景（与 AstrBot LLMPerception 一类做法同向）。
- 中国**法定节假日与调休**依赖 [chinese-workday](https://www.npmjs.com/package/chinese-workday)（国务院放假数据）；未安装依赖时退化为仅按周一～周五 / 周末区分。
- 通过 **IANA 时区**（如 `Asia/Shanghai`）统一时间戳、当地日历日与整点问候触发时刻。

### 时间查询工具

- **dawn_dusk_get_time**：用户询问时间、日期、星期、是否工作日/节假日时调用；支持可选 `timezone` 参数。
- **dawn_dusk_greeting_check**：用户使用「早上好」「晚安」等与时间相关的问候/道别语时调用，返回当前感知行供模型判断。

### 整点主动问候

在设定时刻（默认 0:00、8:00、12:00、18:00）自动发起问候，并根据对话活跃度选择策略：

| 场景 | 判定条件 | 策略 |
|------|---------|------|
| 静默状态 | 距上次交互超过阈值 | 以情景提示触发主动开口问候 |
| 活跃对话中 | 距上次交互很近 | 注入系统提示补丁，在下次回复中自然融入 |
| 半活跃状态 | 介于两者之间 | 延迟等待对话间隙再发送；超时可放弃本次 |

额外判定：**AI 正在播报 TTS 或正在处理用户输入**时，改为注入提示补丁，避免生硬打断。

### 技术说明（chinese-workday 加载）

`chinese-workday` 为 ESM 包，在 Electron 渲染进程若直接动态 `import` 可能不稳定。本插件将 `dist/chinese-workday.cjs.js` 复制为插件目录下的 `.chinese-workday-bundle.cjs` 再 `require`（该缓存文件已列入 `.gitignore`，由运行时生成）。

## 快速开始

1. 将本仓库内容放入 `plugins/community/dawn-dusk-line/` 目录（或保持文件夹名为 `dawn-dusk-line`）。
2. 在插件目录执行 **`npm install`**，以安装 `chinese-workday`。
3. 在插件管理页面启用插件，并按需在配置页调整参数。

可选：安装依赖后可在插件目录执行 `npm run test:workday`，验证节假日模块能否经 CJS 缓存正确加载。

## 配置说明

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| 感知与时钟时区 | IANA 时区名 | `Asia/Shanghai` |
| 注入环境感知到 LLM | 是否在每次请求时注入上述前缀行 | 开启 |
| 启用中国节假日与调休 | 依赖 `npm install`；关闭则仅按周末判断 | 开启 |
| 节假日地区代码 | 目前仅 `CN` 使用国务院数据；其他值退化为仅周末 | `CN` |
| 问候整点时刻 | 24 小时制，逗号分隔，按配置时区 | `0,8,12,18` |
| 静默判定阈值（分钟） | 超过该时间未交互视为静默 | 10 |
| 活跃判定阈值（分钟） | 低于该时间视为活跃 | 3 |
| 延迟等待超时（分钟） | 半活跃时最长等待 | 30 |
| 时间检查间隔（秒） | 轮询整点问候的间隔 | 30 |

## 三级策略流程

```
整点到达
  ├─ AI正在说话/处理中？ → 注入提示词（下次回复自然融入）
  ├─ 用户活跃对话中？    → 注入提示词（不打断对话）
  ├─ 用户已静默？        → 直接主动问候
  └─ 半活跃状态？        → 延迟等待
                           ├─ 等到静默 → 发送问候
                           └─ 超时     → 放弃本次
```

## 仓库更新说明（中文）

### 2026-04-06 — 同步 my-neuro 本地插件 v1.1.0

本次提交与主项目 `live-2d/plugins/community/dawn-dusk-line` 对齐，主要变更如下。

1. **版本与元数据**  
   - 版本号提升至 **1.1.0**（`metadata.json`、`package.json`）。

2. **环境感知与节假日**  
   - 新增可配置的 **时区**、**是否向 LLM 注入环境感知行**、**中国节假日/调休**（`chinese-workday`）及地区代码占位（当前以 `CN` 为准）。  
   - 每次请求可在用户消息前附加「发送时间 + 星期 + 工作日/节假日/调休 + 时段」一行，便于模型理解语境。

3. **工具定义**  
   - `dawn_dusk_get_time` / `dawn_dusk_greeting_check` 的描述与返回值与上述感知行一致，并支持多时区查询参数。

4. **主动问候与稳定性**  
   - 整点策略增加对 **TTS 播放中 / 用户输入处理中** 的检测，避免不当打断。  
   - 通过 `.cjs` 缓存 + `createRequire` 加载 `chinese-workday`，避免在部分 Electron 环境下直接 ESM 动态导入导致的问题。

5. **仓库维护**  
   - 增加 `package.json` / `package-lock.json` 与 `test-load-workday.cjs`，便于依赖安装与本地校验。  
   - `.gitignore` 忽略 `node_modules/` 与运行时生成的 `.chinese-workday-bundle.cjs`。  
   - **未**包含任何 API Key、账号隐私或非本插件内的系统提示词；整点问候相关的**情景/语气类文案**保留在 `index.js` 中，与插件功能一致。
